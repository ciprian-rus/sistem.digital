## Ce este un pattern

Un pattern descrie un flux complet și motivul pentru care este construit într-un anumit fel. Nu este doar o colecție de componente.

Pattern-urile includ:

- nevoia utilizatorului și contextul administrativ;
- pașii fluxului și stările intermediare;
- conținutul recomandat;
- erori, abandon și reluare;
- integrarea autentificării, plăților și notificărilor;
- criterii de accesibilitate și măsurare.

## Primele pattern-uri candidate

1. începerea și reluarea unei cereri;
2. verificarea răspunsurilor înainte de trimitere;
3. confirmarea și numărul de înregistrare;
4. urmărirea stadiului unui dosar;
5. solicitarea unei plăți;
6. notificarea unei decizii.
