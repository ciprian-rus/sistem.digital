# Contribuții la Sistem Digital

Mulțumim pentru interes. Sistem Digital este construit public, iar contribuțiile sunt evaluate după nevoia utilizatorilor, accesibilitate, coerență și mentenabilitate.

## Modalități de contribuție

- raportează o problemă;
- propune o componentă sau un pattern;
- contribuie cu cercetare și exemple;
- îmbunătățește documentația;
- implementează sau testează cod;
- participă la testare de utilizabilitate și accesibilitate.

## Înainte de implementare

Pentru schimbări semnificative, deschide mai întâi un issue sau un RFC. O componentă nouă trebuie să demonstreze:

1. o nevoie reală și repetabilă;
2. imposibilitatea rezolvării adecvate printr-o componentă existentă;
3. comportament accesibil;
4. HTML semantic și progressive enhancement;
5. exemple și documentație;
6. teste automate și, unde este necesar, testare manuală.

## Flux recomandat

1. Alege sau deschide un issue.
2. Discută soluția înainte de schimbări ample.
3. Creează un branch cu nume descriptiv.
4. Adaugă teste și documentație.
5. Deschide un pull request și completează checklist-ul.
6. Răspunde observațiilor de review.

## Conventional commits

Exemple:

- `feat(tokens): add semantic warning colors`
- `fix(button): restore visible focus state`
- `docs(forms): clarify error message guidance`
- `chore(ci): add accessibility checks`

## Definition of Done

O contribuție este completă când:

- criteriile de acceptare sunt îndeplinite;
- build-ul și testele trec;
- accesibilitatea este verificată;
- documentația este actualizată;
- schimbările incompatibile sunt documentate;
- nu introduce dependențe inutile.

## Limba proiectului

Documentația principală este în limba română. API-urile, numele pachetelor și codul folosesc limba engleză pentru interoperabilitate.
